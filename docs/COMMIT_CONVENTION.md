# Commit Convention

## 디렉토리 구조

```
docs/
  PROJECT.md              # 프로젝트 개요, 기술 스택, 결정 사항
  PROCESS.md              # Phase별 체크리스트 및 진행 프로세스
  COMMIT_CONVENTION.md    # 커밋 메시지 규칙 (이 문서)
```

## 커밋 메시지 형식

```
<type>: <subject>

<body (선택)>

Co-Authored-By: <name> <email>
```

### Type

| Type       | 용도                                      |
| ---------- | ----------------------------------------- |
| `feat`     | 새로운 기능 추가                          |
| `fix`      | 버그 수정                                 |
| `chore`    | 빌드, 설정, 패키지 등 기능 외 변경        |
| `refactor` | 기능 변경 없는 코드 구조 개선             |
| `style`    | 코드 포맷팅, 세미콜론 등 (로직 변경 없음) |
| `docs`     | 문서 추가/수정                            |
| `test`     | 테스트 추가/수정                          |

### Subject 규칙

- 한국어 또는 영어 사용 (프로젝트 내 통일)
- 명령형으로 작성 (예: "추가", "수정", "제거")
- 50자 이내

### Body 규칙

- 변경 내용을 `-` 목록으로 나열
- "무엇을" 했는지 간결하게 기술
