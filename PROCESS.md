# HealthyIO - 개발 진행 프로세스

프로젝트 초기 세팅부터 MVP 완성까지의 단계별 진행 프로세스를 정리한다.
중단 후 재개 시 현재 Phase의 체크리스트를 확인하고 미완료 항목부터 이어서 진행한다.

---

## Phase 1: 프로젝트 초기 세팅

### 체크리스트

- [x] Expo + TypeScript 템플릿으로 프로젝트 생성
- [x] Expo Router 설정 (파일 기반 라우팅)
- [x] 디렉토리 구조 생성 (`src/components`, `src/features`, `src/lib`, `src/types`)
- [x] `axios` 설치
- [x] `@tanstack/react-query` 설치
- [x] `zod` 설치
- [x] `expo-camera` 설치
- [x] `expo-image-picker` 설치
- [x] Axios instance 설정 (base URL, interceptor)
- [x] QueryClientProvider 설정
- [x] 앱 빌드 및 실행 확인

### 완료 기준

앱이 빌드되고 빈 화면이 정상 실행된다.

### 재개 시 확인

- 프로젝트가 생성되어 있는가? → 없으면 처음부터
- `node_modules`가 있는가? → 없으면 `npm install` 후 미설치 라이브러리 확인
- 앱이 정상 실행되는가? → 안 되면 에러 로그 확인

## Phase 5: 화면 완성

### 선행 조건

- [ ] Phase 4 API 연동 완료

### 체크리스트

- [ ] 이미지 확인 화면 - 분석 요청 전송 연동
- [ ] 분석 요청 로딩 상태 표시
- [ ] 분석 완료 시 결과 화면 이동
- [ ] 분석 리스트 화면 - API 연동 (날짜순 정렬)
- [ ] 분석 리스트 화면 - 항목 터치 시 상세 이동
- [ ] 분석 리스트 화면 - 빈 상태(empty state) 처리
- [ ] 분석 상세 화면 - Bristol Stool Scale 결과 표시
- [ ] 분석 상세 화면 - 촬영 이미지 표시
- [ ] 분석 상세 화면 - 추가 분석 항목 표시 (확정된 항목만)
- [ ] 전체 플로우 테스트: 촬영 → 요청 → 리스트 → 상세

### 완료 기준

촬영 → 분석 요청 → 리스트 확인 → 상세 보기까지 전체 플로우가 동작한다.

### 재개 시 확인

- 분석 요청이 전송되는가? → 안 되면 mutation 훅 연결 확인
- 리스트에 데이터가 표시되는가? → 안 되면 query 훅 확인
- 상세 화면에 데이터가 표시되는가?
- 전체 플로우가 끊기는 지점이 있는가?

## 진행 순서 요약

```
Phase 1  프로젝트 세팅           ← 프론트 단독
  ↓
Phase 2  내비게이션 & 화면 뼈대  ← 프론트 단독
  ↓
Phase 3  촬영 기능               ← 프론트 단독 + 백엔드 API 스펙 협의 (병렬)
  ↓
Phase 4  API 연동                ← 백엔드 API 필요
  ↓
Phase 5  화면 완성               ← 백엔드 API 필요
  ↓
Phase 6  마무리 & QA
```

Phase 1~3은 백엔드 API 없이 프론트엔드 단독으로 진행 가능하다.
Phase 3 진행 중 백엔드 팀과 API 스펙을 협의하면 Phase 4 진입 시 대기 없이 연동할 수 있다.
